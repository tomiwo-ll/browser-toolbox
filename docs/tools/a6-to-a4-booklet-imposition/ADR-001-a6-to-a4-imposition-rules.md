# ADR-001: A6からA4への冊子面付けルールを定義する

## ステータス

Accepted

## 日付

2026-05-06

## 背景

一般的なA6からA5、A5からA4への面付けはコンビニコピー機の通常機能で提供されている。一方、A6冊子用ページをA4用紙へまとめて面付けし、印刷後に中央裁断して上下の束を作るワークフローは標準機能だけでは扱いづらい。

コピー機印刷時には位置ずれが発生しうるため、A4の中央裁断線を基準にしてずれを吸収しやすい配置にする必要がある。

## 決定

- A6冊子向け画像/PDFを、A4縦の両面印刷用に面付けする。
- 両面印刷は長辺とじ前提とする。
- 右綴じを標準とする。
- 左綴じは右綴じ結果の左右配置を反転して対応する。
- 右綴じの基本配置は以下とする。
  - 表面 上段: `1, N`
  - 表面 下段: `4, N-3`。下段は上下反転。
  - 裏面 上段: `N-1, 2`
  - 裏面 下段: `N-2, 3`。下段は上下反転。
  - 次の紙は `始点 + 4, N - 4`。
- A4用紙を上下に分け、それぞれでA6からA5への面付けを行う。
- 各A6画像は、A4用紙の中央交点に隅が合うように配置する。
- A6枠は 105mm x 148mm とし、画像は元アスペクト比を保つcontain配置にする。
- アスペクト比が合わない場合は画像全体が収まる最大サイズとし、中央寄せし、余白を白で埋める。
- A4の水平中央線と垂直中央線をガイドとして描画する。
- ガイド線は2pt、薄いグレー、実線、画像より背面とする。
- トンボと塗り足しは行わない。
- 入力ページ数は4の倍数に白紙補完する。
- 最終ページを冊子の最終ページに置けるよう、必要なら末尾直前に白紙を挿入する。
- PDF入力は1ファイルのみとし、`pdf.js` でブラウザ内ラスタライズする。
- 暗号化PDF、パスワード付きPDF、巨大PDF、ページサイズ混在PDFは検知して受け入れない。
- 出力形式はPDFまたはZIP(JPG)を選択可能にする。
- 推奨上限は64ページ程度とし、メモリ不足時は分かるメッセージを表示して処理を中断・キャンセルする。

## 採用しなかった案

| 案 | 採用しなかった理由 |
| --- | --- |
| 8の倍数に白紙補完する | A4両面1枚の物理単位とは合うが、11ページ入力で最終画像を12ページ目へ置く要件を満たしにくい。 |
| 左綴じを別アルゴリズムで実装する | 右綴じ結果の左右反転で目的を満たせるため、実装と検証を簡潔にできる。 |
| トンボや塗り足しを追加する | このツールでは中央裁断線を基準にずれを吸収するため、v1では不要。 |
| PDF拒否条件を緩くして可能な範囲で処理する | ブラウザ内処理の安定性とユーザーの印刷事故防止を優先する。 |

## 影響

- 要件定義書、仕様書、トレーサビリティ確認表はこの面付けルールを正本として記述する。
- 実装時には、ページ順生成ロジック、左綴じ反転、白紙補完、ガイド線描画、PDF拒否条件を単体テスト対象にする。
- ツール一覧ではA6からA4への冊子面付けツールとして表示する。
