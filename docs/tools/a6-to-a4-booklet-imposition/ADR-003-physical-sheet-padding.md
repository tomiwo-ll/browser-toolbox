# ADR-003: A4両面シート用に物理スロットを8ページ単位へ補完する

## ステータス

Accepted

## 日付

2026-05-06

## 背景

ADR-002により、ユーザー向けのページ列は4の倍数に白紙補完する方針になった。一方、A4用紙1枚の両面にはA6ページを8スロット配置するため、4の倍数に補完しただけでは最後のA4両面シートが物理的に半端になる場合がある。

例として11ページ入力は、既定では12ページ構成になる。しかしA4両面出力としては2枚分、つまり16スロットが必要になる。この追加分を明示的に扱わないと、面付け式でページ重複や欠落が発生する。

## 決定

- ユーザー向けの論理ページ列は、既定または最終ページ保持オプションに従って4の倍数に補完する。
- 面付け計算と出力生成では、A4両面シートとして成立するよう内部的に8ページ単位へ追加白紙補完する。
- 追加の物理スロット補完は、ユーザー入力ページ順や最終ページ保持オプションの意味を変えない。

## 採用しなかった案

| 案 | 採用しなかった理由 |
| --- | --- |
| すべて8の倍数に補完する仕様へ戻す | ユーザーが求めた4の倍数補完と最終ページ保持オプションの意味が分かりにくくなる。 |
| 半端なA4シートを出力する | 両面印刷と裁断後の作業で扱いにくく、ページ重複や欠落の原因になる。 |

## 影響

- `imposition.ts` は論理ページ補完と物理シート補完を分けて実装する。
- 11ページ入力では、ユーザー向けには12ページ構成として扱い、出力生成では16スロットまで内部補完する。
- トレーサビリティ確認表にREQ-020を追加する。
