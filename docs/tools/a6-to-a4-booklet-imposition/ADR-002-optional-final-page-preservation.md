# ADR-002: 最終ページ保持をユーザー選択オプションにする

## ステータス

Accepted

## 日付

2026-05-06

## 背景

ADR-001では、ページ数が4の倍数でない場合に、最終ページを冊子の最終ページへ置けるよう末尾直前に白紙を挿入する方針を採用した。

しかし、この挙動は入力ページ順のまま面付けしたいユーザーにとっては意図しないページ移動になる。通常は入力順どおりに面付けし、不足分の白紙は最終ページの後ろへ追加されるほうが自然である。一方で、冊子として最終ページを最後に固定したい用途もある。

## 決定

- 既定動作では、ページ順のとおりに面付けし、4の倍数に足りない白紙は入力最終ページの後ろへ追加する。
- 最終ページを冊子の最終ページに置く挙動は、ユーザーが選択できるオプションとして提供する。
- オプションを選択した場合のみ、必要に応じて末尾直前に白紙を挿入する。
- 11ページ入力では、既定動作は11ページ目に入力の最終画像、12ページ目に白紙を配置する。
- 11ページ入力で最終ページ保持オプションを選択した場合は、11ページ目に白紙、12ページ目に入力の最終画像を配置する。

## 採用しなかった案

| 案 | 採用しなかった理由 |
| --- | --- |
| 最終ページ保持を常に有効にする | 入力ページ順どおりの出力を期待するユーザーにとって、暗黙のページ移動になる。 |
| 最終ページ保持機能を廃止する | 冊子の裏表紙や奥付などを最後に固定したい用途を満たせない。 |

## 影響

- 要件定義書のREQ-015は、Mustのオプション提供要件として扱う。
- 仕様書では、白紙補完の既定動作と最終ページ保持オプションを別仕様として定義する。
- トレーサビリティ確認表では、既定補完とオプション補完を別テストで確認する。
- ADR-001の「最終ページ保持」判断は、このADRにより既定動作ではなくオプション動作として扱う。
