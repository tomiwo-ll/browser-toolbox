# ADR-004: ガイド線を画像手前に描画する

## ステータス

Accepted

## 日付

2026-05-06

## 背景

ADR-001では、A4の水平中央線と垂直中央線を2pt薄グレー実線として画像背面に描画する方針にしていた。しかし、裁断・折り位置のガイドは印刷後の物理作業で重要であり、入力画像の上に載ってでも視認できるほうがよい。

このツールで扱うA6冊子向けページは、A4中央のガイド線と重なる位置に読まれるべき内容を置かない前提にできる。

## 決定

- A4の水平中央線と垂直中央線は画像手前に描画する。
- 線幅は2pt、色は薄いグレー、線種は実線を維持する。
- ガイド線は裁断・折り位置を示すためのものであり、入力ページ側ではガイド線と重なる位置に読まれるべき内容を置かない前提とする。

## 採用しなかった案

| 案 | 採用しなかった理由 |
| --- | --- |
| ガイド線を画像背面に描画する | 画像が中央線上にある場合、裁断・折り位置が見えづらくなる。 |
| ガイド線を描画しない | 印刷後の裁断・折り位置をユーザーが判断しづらくなる。 |

## 影響

- `exporters.ts` の描画順を、画像配置後にガイド線を描く順序へ変更する。
- 要件、仕様、トレーサビリティ確認表で、ガイド線が画像手前に描画されることを明記する。
- 既存のADR-001に含まれる「画像より背面」という判断は、このADRにより置き換える。
