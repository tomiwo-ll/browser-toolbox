# 要件定義書: A6 to A4 Booklet Imposition

## メタデータ

- 対象: A6 to A4 Booklet Imposition
- 種別: tool
- 現在バージョン: 0.1.0
- 作成日: 2026-05-06
- 更新日: 2026-05-06
- 関連ADR: ADR-001-a6-to-a4-imposition-rules.md, ADR-002-optional-final-page-preservation.md

## 目的

A6冊子向けの画像またはPDFを、A4両面印刷用にブラウザ内で面付けできるようにする。ユーザーは印刷したA4用紙を中央で裁断し、上下の束を自分で並び替え、ホチキス留めしてA6冊子を作れる。

## 利用者

- コンビニや自宅のコピー機でA6サイズ冊子を印刷したい人。
- 印刷後にカッター、定規、ホチキスなどで裁断・並び替え・製本作業ができる人。
- コピー機の両面印刷、倍率100%、自動拡大縮小なしの設定を行える人。

## 成功条件

| ID | 成功条件 | 確認方法 |
| --- | --- | --- |
| REQ-SC-001 | 16ページ入力で、右綴じ標準の指定面付け順どおりにA4両面2枚分の出力が得られる | 固定入力で出力PDF/ZIP(JPG)を確認する |
| REQ-SC-002 | 左綴じ選択時に、右綴じ標準から左右配置が反転する | 右綴じと左綴じの出力を比較する |
| REQ-SC-003 | 11ページ入力で最終ページ保持オプションを選ばない場合、12ページ構成に補完され、12ページ目が白紙になる | 固定入力でページ配置を確認する |
| REQ-SC-004 | 11ページ入力で最終ページ保持オプションを選んだ場合、12ページ構成に補完され、11ページ目が白紙、12ページ目に最終画像が配置される | 固定入力でページ配置を確認する |
| REQ-SC-005 | JPG/PNG複数入力とPDF単体入力を、外部送信なしで処理できる | ブラウザ内の処理とネットワーク送信なしを確認する |
| REQ-SC-006 | PDFまたはZIP(JPG)を選択して出力できる | 出力形式を切り替えて成果物を確認する |

## 要件

| ID | 要件 | 優先度 | 備考 |
| --- | --- | --- | --- |
| REQ-001 | 入力形式は JPG、PNG、PDF とする | Must | PDFは1ファイルのみ |
| REQ-002 | PDF入力は `pdf.js` でブラウザ内ラスタライズし、PDFページ順を入力順とする | Must | サーバー処理は禁止 |
| REQ-003 | 暗号化PDF、パスワード付きPDF、巨大PDF、ページサイズ混在PDFは検知して受け入れない | Must | 理由が分かるエラーを表示する |
| REQ-004 | 画像入力のページ順はアップロード順とする | Must | アップロード後に確認可能にする |
| REQ-005 | アップロード後にページ順の確認と並び替えができる | Should | MVPで可能な範囲で実装する |
| REQ-006 | 右綴じを標準とし、左綴じを選択可能にする | Must | 左綴じは左右配置反転で対応 |
| REQ-007 | 両面印刷はA4縦・長辺とじ前提とする | Must | UIや説明に明記する |
| REQ-008 | 出力用紙サイズは 210mm x 297mm とする | Must | A4縦 |
| REQ-009 | 出力解像度は300dpi相当とする | Must | ZIP(JPG)出力も同等 |
| REQ-010 | 入力A6画像は 105mm x 148mm を基本枠として扱う | Must | contain配置 |
| REQ-011 | 画像は元アスペクト比を保って最大サイズにリサイズし、中央寄せし、余白を白で埋める | Must | crop/fillは行わない |
| REQ-012 | A4の水平中央線と垂直中央線をガイドとして描画する | Must | 2pt、薄いグレー、実線、画像背面 |
| REQ-013 | トンボと塗り足しは行わない | Must | コピー機ずれを中央裁断線基準で吸収する |
| REQ-014 | 画像枚数は4の倍数に白紙補完する | Must | 既定では最終ページの後ろに白紙を追加する |
| REQ-015 | 最終ページを冊子の最終ページに置けるオプションを用意する | Must | 選択時のみ、11ページなら11ページ目白紙、12ページ目に最終画像 |
| REQ-016 | 出力形式は PDF または ZIP(JPG) を選択できる | Must | ZIP内は面ごとのJPG |
| REQ-017 | 推奨上限は64ページ程度とし、超過または近接時に注意表示する | Should | 処理自体を必ず拒否する要件ではない |
| REQ-018 | メモリ不足時は分かるメッセージを表示し、処理を中断・キャンセルする | Must | ブラウザクラッシュを避ける |
| REQ-019 | すべての処理はブラウザ内で完結し、入力内容を外部へ送信・保存しない | Must | リポジトリ共通方針 |

## 非目標

- コピー機の自動設定、ネットワークプリント連携、外部ストレージ保存は扱わない。
- トンボ、塗り足し、自動ホチキス位置ガイドは扱わない。
- A6以外の入力用紙サイズやA4以外の出力用紙サイズはv1では扱わない。
- 暗号化PDF、パスワード付きPDF、巨大PDF、ページサイズ混在PDFの変換や修復は行わない。

## 制約

- ブラウザ完結ツールでは、入力内容を外部へ送信、保存しない。
- GitHub Pages で配信できる静的サイトとして成立させる。
- ツール単体では、Semantic Versioning に従ってバージョンを管理する。
- 出力PDF/JPGはコピー機側で倍率100%、自動拡大縮小なし、A4縦、長辺とじとして印刷する前提とする。
