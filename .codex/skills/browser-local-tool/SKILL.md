# Browser Local Tool

このスキルは、GitHub Pages 上で動くブラウザ完結ツールを作るときに使う。

## 基本ルール

- ユーザーが投入したファイルはブラウザ内だけで扱う。API、解析サービス、外部ストレージ、サーバー処理へ送信しない。
- 秘密鍵、トークン、認証、バックエンド必須処理、実行時環境変数を追加しない。
- GitHub Pages で配信できる静的ファイルとクライアントサイドコードを前提にする。
- UI、処理ロジック、テスト、テストデータは機能単位で分ける。
- 空入力、不正ファイル、巨大ファイル、リセット、ダウンロード失敗を扱う。
- 重い画像/PDF処理では、Web Worker の採用、メモリ使用量、不要なバッファ複製の削減を検討する。

## 実装チェックリスト

1. `docs/features/<yyyy-mm-dd>-<feature-slug>/` に要件、ADR、仕様、突合表を作る。
2. 入力ファイルが外部送信されず、成果物がブラウザ内で生成されることを確認する。
3. パース、検証、変換、ユーザーに見える失敗状態のテストを追加する。
4. lint、typecheck、test、build は Docker 経由で実行する。
5. PR本文に検証コマンドと残リスクを書く。
